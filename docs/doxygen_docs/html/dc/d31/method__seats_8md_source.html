1 ## Method : seats
2 
3 Once the DCV rceives the game information, it request the information about the seats on the table. DCV sents the below JSON message to the backend to get the seat information
4 ```json
5 {"method":"seats"}
6 ```
7 The response from backend is shown below
8 ```json
9 {
10  "method": "seats",
11  "seats": [{
12  "name": "player1",
13  "seat": 0,
14  "stack": 0,
15  "empty": 0,
16  "playing": 1
17  }, {
18  "name": "player2",
19  "seat": 1,
20  "stack": 0,
21  "empty": 0,
22  "playing": 1
23  }]
24 }
25 ```
26 The significance of each field is mentioned as follows:
27 * __name__ : It referes to the name of the player, I hardcoded these values to **player1** and **player2**
28 * __seat__ : This value referes to the relative positioning of the player on the table. The front end GUI which `pangea-poker-frontend` has
29 max table size of **9**.
30 * __stack__: This value refers to the amount of funds that each player is having at the begining of the game.
31 * __empty__: I didn't used this field abywhere in the frontend GUI, can be considered optional.
32 * __playing__: This field is used to make the seat available for playing, so tha any players can chose the seat with with value **1** and join the table.
