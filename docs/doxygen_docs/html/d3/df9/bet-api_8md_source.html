1 # Usage guide of the bet-api's
2 ## Create a player
3 In the context of the game the identity of the player is a point on the Curve25519 which is known to every participating entity in the game.

13 ## Create a deck
14 Every asset in the game is a point the Curve25519, whether it could a card in the deck, a number in the dice or a side of the coin. In the context of the game the identity of an asset is only revealed during the game when and where it is needed.
15 ```
16 ./bet-cli create-deck numOfCards
17 For example: ./bet-cli create-deck 2(sides on a coin or a deck of two cards numbered 0-1)

34 ## Blind Deck
35 This player takes the deck as input and blinds the privkey of deck by multiplying it with the pubkey of the player. This way a player can conceal the identity of the deck.

61 ## Player Join Request
62 This API allows the player to make join request to the Deck Creating Vendor(Dealer) and in response it gets peerid assigned by DCV. The prerequisite for this is to make sure a Dealer is already up and running to get the response.
63 ```
64 ./bet-cli join-req playerPubKey srcBindAddr destBindAddr
65 
66 In the below example player node and dealer node are communicating using IPC.
67 
68 ./bet-cli join-req "{\n\t\"command\":\t\"create-player\",\n\t\"PubKey\":\t\"5094b1e04e91b16d1a099fcaa25f5618c912a0b53cd196c16050875843eb095f\"\n}" "ipc:///tmp/bet.ipc" "ipc:///tmp/bet1.ipc"
69 nntype.80 connect to ipc:///tmp/bet1.ipc connectsock.1
70 nntype.33 connect to ipc:///tmp/bet.ipc connectsock.2
